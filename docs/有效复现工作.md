### 分类说明

- **直接收录**：来源可信（官方或多方交叉验证）、方法透明、无泄露嫌疑。MiniRocket 基线经官方+标答A+标答B 三方对齐，但注意官方 tsai 实现与 sktime 实现是**同名不同命的两种算法**。
- **待审计**：工作本身完整（有代码有数字），但存在以下一种或多种情况：(1) 数字异常高需验证；(2) 多阶段融合可能引入泄露；(3) 关联已撤稿论文需排查缺陷；(4) 与官方基准差距大需确认原因；(5) 无代码但分析有价值。需亲自跑通或逐行审查代码。
- **待补充**：代码/框架在，但缺最终指标（Kaggle 系列），或任务不对齐（IJCNN SOM）。补全数字或改写任务后可升级。
- **附录**：结果不可比（硬件限制、协议冲突等），但作为领域生态观察记录。

### 重要更新（2026-7-4）

1. **数据增强禁令**：AMBC Standard Track 禁止 Sliding Window、Window Slice、TimeWarp、Gaussian Noise 等任何训练时数据增强。统一协议下，一个航班 = 一个样本，预处理阶段完成后不再对训练数据进行变形。
2. **MiniRocket 实现分裂**：官方 0.598 为 tsai 路线（PyTorch GPU + 神经网络头），AMBC 自跑 0.7246 为 sktime 路线（CPU + RidgeClassifierCV）。两者共享名字但算法不同，不可直接比较。
3. **官方 ConvMHSA 被低估**：知乎 aq-void 分析指出官方 TF 实现存在四大工程缺陷（无BN、无位置编码、无动态学习率、ReLU），规范实现（PyTorch+BN+GELU）可达 0.775+，高于官方 0.760。
